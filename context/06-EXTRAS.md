## Présentation projet (PulsaBot)

> PulsaBot est un bot Discord musical que j'ai développé. Il utilise Lavalink pour le streaming, supporte Spotify et Deezer, et inclut un dashboard React pour gérer la config des serveurs, les playlists et les stats. Le projet est conteneurisé avec Docker et déployé via GitHub Actions. Il compte aussi des modules communautaires : niveaux, giveaways, tickets, modération.

## Réponses types entretien

### "Parle-moi de ton projet le plus complexe"

> PulsaBot combine un bot Discord (Node.js, discord.js), une API REST (Express), un dashboard React avec OAuth Discord, et une base SQLite. La partie la plus technique était l'intégration Lavalink pour le streaming audio multi-source (Spotify, Deezer), et le déploiement automatisé avec Docker et GitHub Actions. J'ai aussi mis en place une documentation structurée (.context) pour maintenir le projet sur la durée.

### "Quelles technologies utilises-tu ?"

> Backend : Node.js, Express, SQLite. Frontend : React 19, TypeScript, Vite. Pour PulsaBot : discord.js, Lavalink, Docker. Je travaille avec Git, GitHub Actions pour la CI/CD, et j'écris des tests avec Vitest.

### "Comment gères-tu la documentation ?"

> J'utilise un dossier .context avec des fichiers markdown numérotés (overview, stack, API, etc.). Chaque fichier couvre un aspect du projet. Ça permet de restaurer le contexte rapidement, de partager avec une IA, ou d'onboarder quelqu'un. J'ai appliqué le même principe à mon portfolio.
